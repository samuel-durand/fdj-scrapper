📌 CRÉER LE FICHIER
───────────────────────────────────────────────────────────────

1. À la RACINE du projet, créer le fichier : .env.production

2. Y mettre ce contenu :


═══════════════════════════════════════════════════════════════
  📄 CONTENU DU FICHIER .env.production
═══════════════════════════════════════════════════════════════

# URL de l'API backend en production
VITE_API_URL=https://api.resultat-fdj.soqe8286.odns.fr/api


═══════════════════════════════════════════════════════════════
  🎯 OPTIONS POSSIBLES
═══════════════════════════════════════════════════════════════

Choisir UNE option selon ta configuration :

Option 1 : Sous-domaine o2switch → Render (RECOMMANDÉ)
────────────────────────────────────────────────────────────
VITE_API_URL=https://api.resultat-fdj.soqe8286.odns.fr/api


Option 2 : URL Render directe (sans sous-domaine)
────────────────────────────────────────────────────────────
VITE_API_URL=https://loterie-backend.onrender.com/api


Option 3 : VPS personnel
────────────────────────────────────────────────────────────
VITE_API_URL=https://api.mondomaine.com/api


Option 4 : Test local du build
────────────────────────────────────────────────────────────
VITE_API_URL=http://localhost:5000/api


═══════════════════════════════════════════════════════════════
  ⚠️  NOTES IMPORTANTES
═══════════════════════════════════════════════════════════════

✅ Toujours utiliser HTTPS en production
✅ L'URL DOIT se terminer par /api
✅ PAS de slash final après /api
❌ Ne JAMAIS commit .env.production sur Git (déjà dans .gitignore)


═══════════════════════════════════════════════════════════════
  🚀 APRÈS CRÉATION
═══════════════════════════════════════════════════════════════

1. Vérifier que le fichier .env.production existe à la racine

2. Builder le frontend :
   npm run build

3. OU utiliser le script :
   scripts\build-frontend-o2switch.bat


═══════════════════════════════════════════════════════════════
  ✅ VÉRIFICATION
═══════════════════════════════════════════════════════════════

Après le build, tester localement :

npm run preview

Ouvrir http://localhost:4173 et vérifier dans la console (F12)
que les appels API vont vers la bonne URL.
